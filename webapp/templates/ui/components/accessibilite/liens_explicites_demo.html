A11Y-13 — Intitulés de liens explicites

Conformément à RGAA 6.1, l'intitulé d'un lien doit être pertinent et reprendre le texte visible quand un title est utilisé.

1. Lien des logos du header

Comme les 3 SVG enfants sont désormais aria-hidden="true" (cf. A11Y-11), le lien parent n'avait plus aucun contenu textuel accessible. Ajout d'un <span class="qf-sr-only"> qui reprend le title. 

<a href="/" title="Accueil — Que faire de mes objets et déchets">
  <span class="qf-sr-only">Accueil — Que faire de mes objets et déchets</span>
  <svg aria-hidden="true" .../>
  ...
</a>

2. Lien « disponible librement » dans la modale d'intégration

Avant : title="Lien vers le dépôt GitHub - Nouvelle fenêtre" (ne reprend pas l'intitulé visible).

Après : title="disponible librement - Nouvelle fenêtre" (reprend le texte du lien et signale l'ouverture dans une nouvelle fenêtre).

L'outil est développé de manière ouverte (open source). L'ensemble du code est disponible librement.
